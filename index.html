The Goal

I wanted to build an app that helps dog owners see how busy a park is in real-time. The goal was to move away from static reviews and create a live "check-in" system where users can see exactly which dogs are at the park right now.

I used Supabase for the backend because it handles real-time updates easily, and PostgreSQL with PostGIS to manage all the map coordinates efficiently.

Technical Breakdown

1. Fixing Map Clusters

When dogs are at the same spot, their markers stack on top of each other. I wrote a small "jitter" function that uses math to nudge the icons slightly so you can see and click each one individually.

2. Real-Time Chat

I used Supabase listeners to build a live chat. This lets owners talk to people currently at the park to check things like the weather or the current "vibe" before heading out.

3. Custom Dog Profiles

Users can set up their dog's breed and temperament. I built the database so these traits are linked to the check-in, helping other owners know if the current group of dogs is a good fit for their pet.

4. Simple & Responsive UI

I focused on making the app feel snappy. I used React Native hooks to manage the app state and added haptic feedback (vibrations) so the app feels more tactile when you check in.
