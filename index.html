Problem & Objective

The primary engineering challenge was to create a low-latency, event-driven map interface that provides real-time park occupancy data. Unlike typical social apps, the data here is ephemeral; check-ins must be accurate within seconds and expire automatically to maintain system integrity.

I architected the backend using Supabase (PostgreSQL) to leverage native spatial indexing, ensuring that querying hundreds of live coordinates remains performant on mobile hardware.

System Architecture

Frontend Client

React Native (TypeScript) managing geolocation via expo-location and global state via Hooks.

Spatial Logic

Supabase RPC (Remote Procedure Calls) calculating distances using PostGIS on the server side.

Sensory Layer

Asynchronous UX feedback loops utilizing expo-av and native Haptic engines.

Development Deep-Dive

1. Geofencing & Pin Optimization

Implemented a "jitter" algorithm to spread markers that share identical GPS coordinates, preventing UI stacking in popular areas.

  • + Optimized PostGIS spatial queries
  • + Vibe-based conditional rendering

2. Real-Time Messaging State

Utilized supabase-js listeners to create a live message board. Messages are strictly coupled with the active "check-in" session to ensure community relevance.

3. Dynamic Pet Logic Layer

Developed a robust metadata system for dog profiles. This allows for conditional UI rendering based on temperament, and activity level, providing a safer environment for users.

  • + Relational PostgreSQL Schema
  • + Dynamic Icon Mapping

4. Native UX Components

Focusing on high-fidelity mobile performance, I utilized React Native FlatLists for smooth message scrolling and integrated native Haptic feedback for every successful park check-in.
